What to expect

Behavior
A colony of artificial ants cooperatively searches for the shortest roundtrip through all of the data points in an input file.
Input
A TSPLIB file containing x-y coordinates or distances between points.
Output
Statistics about the best (shortest) tour found, statistics comparing each iteration of the colony, and internal statistics that are not labeled.

Quickly try it out

First, list the built-in scripts and choose one to run. 

$ demo run

djibouti      Find the shortest roundtrip through Djibouti (5s)
                  38     cities
                  6656   optimal tour

qatar         Find the shortest roundtrip through Qatar (5s)
                  194    cities
                  9532   optimal tour

luxembourg    Find the shortest roundtrip through Luxembourg (5s)
                  980    cities
                  16342  optimal tour
Next to the name of each script in parentheses is how long the script likely takes to complete (5s = 5 seconds)

Then, run the script. 

$ demo run djibouti

The demo solves the traveling salesman problem heuristically, so you may not always get the optimal result.

Experiment with options

First, find command line options in the script output. 

[Command line]

/root/src/acotsp \
    -i /root/data/djibouti/dj38.tsp \
    --tries 1 \
    --time 5
Find all of the command line options in the run reference.

Then, print a description to learn how they work. 

$ demo run help -i

Choose a TSPLIB formatted file to use as input to the demo (required)

Included in the demo *:

    Cities of Djibouti    /root/data/djibouti/dj38.tsp
    Cities of Qatar       /root/data/qatar/qa194.tsp
    Cities of Luxembourg  /root/data/luxembourg/lu980.tsp

* = From "National Traveling Salesman Problems", University of Waterloo

Finally, use the options with demo run, changing ones that interest you. 

$ demo run -i /root/data/qatar/qa194.tsp --tries 1 --time 10

A Use new data

You can use any dataset that can be represented as a distance-based graph. For example, the Van Gogh dataset from Robert Bosch traces out a portrait of Van Gogh the artist when the optimal tour is found.

See TSP Art Instances for more TSPLIB files like this

This is a snippet of the Van Gogh dataset in TSPLIB format. It has 120,000 x-y coordinates.

A Trace and debug

First, find [Command line] in the lines printed by the script into your shell. 

[Command line]

/root/src/acotsp \
    -i /root/data/djibouti/dj38.tsp \
    --tries 1 \
    --time 5

Copy and paste them into your shell and execute them. Do this to make sure that running the demo manually works before adding extra tools. 

$ /root/src/acotsp -i /root/data/djibouti/dj38.tsp --tries 1 --time 5

Then, install a tracing or debugging program. For example, if you're interested in seeing which files the demo uses, install strace. 

$ apt-get install strace

Finally, wrap the commands you tested in your tracing or debugging program. 

$ strace -f -e open \
/root/src/acotsp -i /root/data/djibouti/dj38.tsp --tries 1 --time 5
