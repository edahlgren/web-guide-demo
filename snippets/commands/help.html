Layout of the demo

/root/src Source code
/root/data Data sets
/shared Can be synchronized with a local directory
/demo/setup Scripts used to setup the demo
/demo/run Scripts for running the demo
/demo/build Scripts for building the demo
/demo/docs Generated docs
/demo/demo.yml Configuration file that fully describes the demo

Experimentation guide

Getting started

The easiest way to experiment with the demo is inside the demo shell, a terminal environment with everything needed by the demo already installed. Inside it, try each of the configured run scripts to test what the demo can do. Then try changing the behavior of the demo by using different command line options and input data.

Digging deeper

Once you've seen what the demo can do, you can learn how it works by reading its documentation, research papers, and source code. You can also rebuild it after changing the source code to test out new ideas. When you have changes to save, add new content to the demo like data sets, scripts, command line options, and source code files, and regenerate the documentation.
